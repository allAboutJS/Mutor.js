# Code of Conduct

## Our Standard

Mutor.js is a small, focused open-source project. Participation — whether through issues, pull requests, or discussions — is expected to be professional and respectful.

Concretely, this means:

- Engage with ideas, not people. Critique code and design decisions directly; do not make it personal.
- Be precise. Vague complaints and demands are not useful. Specific, reproducible issues and reasoned arguments are.
- Assume good faith unless clearly demonstrated otherwise.
- Accept that maintainers make final calls on design and scope. Disagreement is fine; sustained pressure after a decision has been made is not.

## What Is Not Acceptable

- Harassment, insults, or personal attacks in any form
- Dismissiveness toward contributors because of their experience level, background, or identity
- Deliberate misrepresentation of others' positions
- Conduct that would not be acceptable in a professional engineering setting

## Scope

This standard applies in all project spaces: GitHub issues, pull requests, discussions, and any other venue where Mutor.js is being discussed in a capacity related to the project.

## Enforcement

Violations can be reported to the maintainer, **Onah Victor**, via GitHub direct message or private vulnerability reporting. Reports will be handled discreetly.

Consequences range from a warning to a permanent ban from the project, depending on the severity and pattern of the behavior.

---

*This code of conduct is intentionally concise. It does not try to enumerate every possible situation — it establishes a clear standard and trusts contributors to apply it.*
